# CLI Reference

The `hambros` CLI provides command-line access to local install setup, startup, agent onboarding, quest management, worker dispatch, memory operations, and cron scheduling.

## Installation

The CLI is included in the HamBros monorepo as `@hambros/cli`. The recommended install path is:

```bash
curl -fsSL https://raw.githubusercontent.com/NickGuAI/HamBros/main/install.sh | bash
```

If you are working from a source checkout instead, build the workspace dependencies first:

```bash
# From the HamBros root
pnpm --filter hambros run build:deps
```

The CLI binary is available at `packages/cli/bin/hambros.mjs`.

## Commands

---

### `hambros init`

Interactive local setup for a self-hosted HamBros install.

```bash
hambros init
```

The wizard:
- Detects whether `~/.hambros/app/.env` already exists
- Prompts for the server port and admin API key
- Lets you choose which local agents to instrument
- Writes `app/.env` and `~/.hambros.json`
- Prints the next step: `hambros start`

---

### `hambros start`

Launches HamBros from the installed app root, not from your current shell directory.

```bash
hambros start
```

When a production client build exists, `hambros start` runs the source server with `NODE_ENV=production` so the built UI is served correctly at `http://localhost:20001`. If compiled server artifacts are the only available entrypoint, it falls back to the built server.

---

### `hambros onboard`

Interactive setup wizard that configures AI agents to send telemetry to your HamBros instance.

```bash
hambros onboard
```

The wizard prompts for:
- **HamBros endpoint** — URL of your HamBros instance (default: `http://localhost:20001`)
- **API key** — authentication key
- **Agent selection** — which agents to configure (multi-select)

Supported agents:
- Claude Code (auto-configured)
- Codex (auto-configured)
- Cursor (auto-configured)
- Terminal CRI (manual setup)
- Anti-Gravity (manual setup)

Configuration is saved to `~/.hambros.json`.

### `hambros quests`

Manage the commander quest board.

> Requires `HAMMURABI_COMMANDER_ID` environment variable for `list` operations.

#### `quests list`

```bash
HAMMURABI_COMMANDER_ID=<uuid> hambros quests list
```

List pending and active quests.

#### `quests create`

```bash
hambros quests create \
  --instruction "Fix the login page redirect bug" \
  --cwd /path/to/project \
  --mode default \
  --agent claude
```

| Flag | Required | Description |
|------|----------|-------------|
| `--instruction` | Yes | Task description |
| `--cwd` | Yes | Working directory |
| `--mode` | Yes | Permission mode |
| `--agent` | Yes | Agent type (`claude`, `codex`) |
| `--skills` | No | Comma-separated skill list |
| `--source` | No | Origin identifier |
| `--issue` | No | GitHub issue URL |
| `--note` | No | Initial note |

#### `quests delete <id>`

Delete a quest by ID.

#### `quests claim <id>`

Mark a quest as active.

#### `quests note <id> "<text>"`

Add a progress note to a quest.

#### `quests done <id> --note "<text>"`

Mark a quest as completed with a completion note.

#### `quests fail <id> --note "<text>"`

Mark a quest as failed with a failure note.

#### `quests artifact add <id>`

```bash
hambros quests artifact add <quest-id> \
  --type github_pr \
  --label "Fix PR" \
  --href https://github.com/org/repo/pull/43
```

Artifact types: `github_issue`, `github_pr`, `url`, `file`

#### `quests artifact remove <id> <href>`

Remove an artifact from a quest.

### `hambros workers`

Manage factory worker sessions.

#### `workers list`

List active factory workers.

#### `workers dispatch`

```bash
hambros workers dispatch \
  --session main-session \
  --issue https://github.com/org/repo/issues/42 \
  --branch fix-issue-42 \
  --task "Fix the login redirect bug" \
  --agent claude
```

| Flag | Required | Description |
|------|----------|-------------|
| `--session` | Yes | Parent session name |
| `--issue` | Yes* | GitHub issue URL |
| `--branch` | Yes* | Feature branch name |
| `--task` | No | Task description |
| `--machine` | No | Target machine ID |
| `--agent` | No | Agent type (default: `claude`) |

*At least one of `--issue` or `--branch` is required.

#### `workers kill <name>`

Kill a worker session by name.

#### `workers status <name>`

Get worker session status.

#### `workers send <name> "<text>"`

Send a message to a running worker.

### `hambros memory`

Manage commander memory.

#### `memory find`

```bash
hambros memory find --commander <id> "authentication redirect"
```

| Flag | Required | Description |
|------|----------|-------------|
| `--commander` | Yes | Commander ID |
| `--top` | No | Number of results to return |

#### `memory save`

```bash
hambros memory save --commander <id> "Auth redirects require URL encoding"
```

Multiple facts can be saved:

```bash
hambros memory save --commander <id> \
  "Auth redirects require URL encoding" \
  --fact "Login page uses /callback as redirect path"
```

#### `memory compact`

```bash
hambros memory compact --commander <id>
```

Consolidates memory, extracts facts, and prunes stale entries.

### `hambros cron`

Manage scheduled tasks.

#### `cron list`

```bash
hambros cron list --commander <id>
```

#### `cron add`

```bash
hambros cron add \
  --commander <id> \
  --schedule "0 9 * * *" \
  --instruction "Review open PRs" \
  --agent claude \
  --name daily-review
```

| Flag | Required | Description |
|------|----------|-------------|
| `--commander` | Yes | Commander ID |
| `--schedule` | Yes | Cron expression |
| `--instruction` | Yes | Agent prompt |
| `--name` | No | Task name |
| `--agent` | No | Agent type |
| `--session-type` | No | `stream` or `pty` |
| `--permission-mode` | No | Tool approval behavior |
| `--work-dir` | No | Working directory |
| `--machine` | No | Target machine |
| `--disabled` | No | Create in disabled state |

#### `cron update <id>`

```bash
hambros cron update <cron-id> \
  --schedule "0 */6 * * *" \
  --enabled false
```

#### `cron delete`

```bash
hambros cron delete --commander <id> <cron-id>
```

#### `cron trigger`

```bash
HAMMURABI_COMMANDER_ID=<uuid> hambros cron trigger \
  --instruction "Run the daily report now"
```

### `hambros commander`

Initialize and configure commander instances.

#### `commander init`

```bash
hambros commander init
```

Interactive setup that creates a local `COMMANDER.md` configuration.

#### `commander init --remote`

```bash
hambros commander init \
  --remote https://your-server.com \
  --token sync-token-123 \
  --commander <id> \
  --poll-interval 30
```

Initialize with remote sync to a HamBros server.
