# Deployment

HamBros can be self-hosted on any machine with Node.js 20+ or deployed to cloud platforms.

## Self-Hosting

### Prerequisites

- Node.js 20+
- pnpm 9+
- A Linux or macOS server (Windows via WSL)

### Recommended Install

```bash
curl -fsSL https://raw.githubusercontent.com/NickGuAI/HamBros/main/install.sh | bash
hambros init
```

This installs HamBros into `~/.hambros` by default, builds the app, and installs the `hambros` CLI into `~/.local/bin/hambros`.

### Start the Server

```bash
hambros start
```

The server listens on port `20001` by default and serves the built UI from the same process.

### Manual Source Build

If you need a checkout-managed deployment instead of the installer-managed layout:

```bash
git clone https://github.com/NickGuAI/HamBros.git
cd HamBros
pnpm install
pnpm --filter hambros run build
```

This produces:
- `app/dist/` — compiled frontend (static assets)
- `app/dist-server/` — compiled backend (Node.js)

### Running with PM2

```bash
npm install -g pm2

# Start the server
pm2 start "$HOME/.local/bin/hambros" --name hambros -- start

# Auto-restart on reboot
pm2 startup
pm2 save
```

If your Node.js or pnpm binaries live outside the default system `PATH`, add the required paths in your PM2 startup environment before enabling auto-start.

### Running with systemd

Create `/etc/systemd/system/hambros.service`:

```ini
[Unit]
Description=HamBros Agent Observability
After=network.target

[Service]
Type=simple
User=hambros
WorkingDirectory=/home/hambros
Environment=HAMBROS_HOME=/home/hambros/.hambros
Environment=PATH=/home/hambros/.local/bin:/usr/local/bin:/usr/bin:/bin
ExecStart=/home/hambros/.local/bin/hambros start
Restart=on-failure
RestartSec=10
Environment=PORT=20001
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable hambros
sudo systemctl start hambros
```

### Environment Variables

For production, set at minimum:

```env
PORT=20001
HAMBROS_ALLOWED_ORIGINS=https://your-domain.com
HAMBROS_SETTINGS_ENCRYPTION_KEY=<random-32-char-string>
NODE_ENV=production
```

See [Configuration](/docs/getting-started/configuration) for all options.

### Reverse Proxy (nginx)

```nginx
server {
    listen 80;
    server_name hambros.example.com;

    location / {
        proxy_pass http://127.0.0.1:20001;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_read_timeout 86400;
    }
}
```

The `proxy_read_timeout` and WebSocket upgrade headers are required for the real-time streaming features.

## Data Directory

HamBros stores runtime data in `app/data/`:

```
app/data/
├── agents/         # Session metadata
├── api-keys/       # Encrypted API keys
├── commanders/     # Commander configurations
├── command-room/   # Task definitions and run history
└── telemetry/      # Telemetry JSONL store
```

For the default installer layout, this directory lives under `~/.hambros/app/data/`. Back it up regularly. It is gitignored by default.

## Vercel (Docs Site Only)

The documentation site (this site) is deployed on Vercel:

- **Root Directory**: `docs`
- **Framework**: Next.js
- **Build Command**: `next build`
- **Output Directory**: `.next`
- **Domain**: `hambros.gehirn.ai`

The `vercel.json` in the `docs/` directory configures the deployment.

> The main HamBros app requires a persistent server process (for WebSocket, node-pty, cron) and cannot run on serverless platforms like Vercel.

## Security Checklist

Before deploying to production:

- [ ] Delete the master key file (`app/data/api-keys/master-key.json`)
- [ ] Create scoped API keys for each consumer
- [ ] Set `HAMBROS_SETTINGS_ENCRYPTION_KEY` to a strong random value
- [ ] Configure `HAMBROS_ALLOWED_ORIGINS` to your domain only
- [ ] Set up HTTPS via reverse proxy or load balancer
- [ ] Configure Auth0 for team access (optional)
- [ ] Set up regular backups of `app/data/`
